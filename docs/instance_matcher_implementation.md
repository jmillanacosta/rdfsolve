# Instance Matcher — Implementation Plan

> **Goal**: Given a bioregistry resource prefix (e.g. `ensembl`), query every rdfsolve endpoint for classes whose instances match the resource's known URI patterns, then generate JSON-LD mapping documents with `skos:narrowMatch` edges between those classes. These mapping documents get imported into the database alongside schemas, populating the graph with new cross-dataset edges the frontend can walk.

---

## 0. Architecture Fit

The instance matcher is a **new data-ingestion pathway** that sits beside the existing miner:

```
                       data/sources.csv
                            │
               ┌────────────┴────────────┐
               │                         │
        SchemaMiner                InstanceMatcher       ◄── NEW
        (miner.py)                 (instance_matcher.py)
               │                         │
        MinedSchema              InstanceMapping          ◄── NEW
        (models.py)              (models.py)
               │                         │
        .to_jsonld()             .to_jsonld()
               │                         │
               └──────────┬──────────────┘
                          │
                   docker/schemas/          (or docker/mappings/)
                   *.jsonld files
                          │
                   SchemaService.import_from_directory()
                          │
                   SQLite DB  →  Frontend  →  walkable paths
```

The miner produces **intra-dataset** schemas. The instance matcher produces **inter-dataset** mappings. Both are JSON-LD with `@context`/`@graph`/`@about` and both feed into the same database → frontend pipeline.

### 1d. Where to add the `predicate` flexibility

`MappingEdge.predicate` defaults to `skos:narrowMatch` but the caller can override it at the `from_bioregistry_resource` level. Later, the UI or CLI can offer a dropdown: `skos:narrowMatch`, `skos:broadMatch`, `skos:exactMatch`, `skos:closeMatch`, `owl:sameAs`, `rdfs:seeAlso`.

## 2. SPARQL Helper — `src/rdfsolve/sparql_helper.py`

Add **one** new convenience method to `SparqlHelper`:

```python
def find_classes_for_uri_pattern(self, uri_pattern: str) -> list[str]:
    """Find all rdf:type classes whose instances match a URI pattern.

    Executes:
        SELECT DISTINCT ?c WHERE {
            ?s a ?c .
            FILTER(STRSTARTS(STR(?s), "uri_pattern"))
        }

    Args:
        uri_pattern: URI prefix string, e.g. "https://identifiers.org/ensembl/"

    Returns:
        List of class URIs.
    """
    query = (
        'SELECT DISTINCT ?c WHERE { '
        '?s a ?c . '
        f'FILTER(STRSTARTS(STR(?s), "{uri_pattern}")) '
        '}'
    )
    out = self.select(query)
    bindings = out.get("results", {}).get("bindings", [])
    return [b["c"]["value"] for b in bindings if "c" in b]
```

> **Note**: Use `STRSTARTS` instead of `regex` — it's orders of magnitude faster on most SPARQL engines and it's all we need since bioregistry gives us URI _prefixes_, not patterns with wildcards.

## 3. Instance Matcher module — `src/rdfsolve/instance_matcher.py`

This is the **core engine**. Keep it as a pure-library module (no Flask, no Click) so it can be used everywhere.

```python
"""Instance-based matching: probe endpoints for bioregistry URI patterns.

Produces InstanceMapping objects that can be serialized to JSON-LD
and imported into the rdfsolve database alongside mined schemas.
"""

from __future__ import annotations

import logging
from typing import Optional

import bioregistry
import pandas as pd

from rdfsolve.models import (
    AboutMetadata,
    InstanceMapping,
    InstanceMatchResult,
    MappingEdge,
)
from rdfsolve.sparql_helper import SparqlHelper

logger = logging.getLogger(__name__)


def probe_resource(
    prefix: str,
    datasources: pd.DataFrame,
    predicate: str = "http://www.w3.org/2004/02/skos/core#narrowMatch",
    dataset_names: Optional[list[str]] = None,
    timeout: float = 60.0,
) -> InstanceMapping:
    """Probe endpoints for a bioregistry resource.

    Steps:
        1. Look up URI formats from bioregistry for `prefix`
        2. For each dataset in datasources (or the subset in dataset_names):
           a. Create SparqlHelper for the endpoint
           b. For each URI format, call find_classes_for_uri_pattern()
           c. Record InstanceMatchResult for each hit
        3. Generate MappingEdges: for every pair of (dataset_A.class, dataset_B.class)
           that both matched the same resource, create a skos:narrowMatch edge
        4. Return InstanceMapping

    Args:
        prefix: Bioregistry prefix (e.g. "ensembl")
        datasources: DataFrame with at least [dataset_name, endpoint_url]
        predicate: Mapping predicate URI
        dataset_names: Optional filter — only probe these datasets
        timeout: SPARQL query timeout in seconds

    Returns:
        InstanceMapping ready for .to_jsonld() export
    """
    # 1. Get URI formats
    resource = bioregistry.get_resource(prefix)
    if resource is None:
        raise ValueError(f"Unknown bioregistry prefix: {prefix}")
    uri_formats = [fmt.rstrip("$1") for fmt in resource.get_uri_formats()]

    # 2. Filter datasources
    df = datasources.copy()
    if dataset_names:
        df = df[df["dataset_name"].isin(dataset_names)]

    # 3. Probe
    match_results: list[InstanceMatchResult] = []
    for _, row in df.iterrows():
        dataset = row["dataset_name"]
        endpoint = row["endpoint_url"]
        sparql = SparqlHelper(endpoint, timeout=timeout)

        for uri_format in uri_formats:
            try:
                classes = sparql.find_classes_for_uri_pattern(uri_format)
            except Exception as exc:
                logger.warning("Probe failed: %s / %s — %s", dataset, uri_format, exc)
                continue

            for cls_uri in classes:
                match_results.append(InstanceMatchResult(
                    dataset_name=dataset,
                    endpoint_url=endpoint,
                    uri_format=uri_format,
                    matched_class=cls_uri,
                ))

    # 4. Generate edges: every pair of classes across different datasets
    #    that matched the same resource → one edge
    hits = [r for r in match_results if r.matched_class]
    edges: list[MappingEdge] = []
    seen: set[tuple[str, str]] = set()
    for i, a in enumerate(hits):
        for b in hits[i + 1:]:
            if a.dataset_name == b.dataset_name:
                continue
            key = tuple(sorted([
                f"{a.dataset_name}:{a.matched_class}",
                f"{b.dataset_name}:{b.matched_class}",
            ]))
            if key in seen:
                continue
            seen.add(key)
            edges.append(MappingEdge(
                source_class=a.matched_class,
                target_class=b.matched_class,
                predicate=predicate,
                source_dataset=a.dataset_name,
                target_dataset=b.dataset_name,
            ))

    # 5. Assemble
    about = AboutMetadata.build(
        dataset_name=f"{prefix}_instance_mapping",
        strategy="instance_matcher",
        pattern_count=len(edges),
    )

    return InstanceMapping(
        edges=edges,
        about=about,
        resource_prefix=prefix,
        uri_formats=uri_formats,
        match_results=match_results,
    )
```

```python
def probe_instance_mapping(
    prefix: str,

def seed_instance_mappings(
    prefixes: List[str],
    sources_csv: str = "data/sources.csv",
    output_dir: str = "docker/schemas",
    predicate: str = "http://www.w3.org/2004/02/skos/core#narrowMatch",
    dataset_names: Optional[List[str]] = None,
) -> Dict[str, Any]:
    """Probe multiple resources and write mapping JSON-LD files to disk.

    Same pattern as mine_all_sources: iterate prefixes, write
    {prefix}_instance_mapping.jsonld to output_dir.

    Returns:
        Summary dict with "succeeded" and "failed" lists.
    """
    from rdfsolve.instance_matcher import probe_resource

    datasources = pd.read_csv(sources_csv)
    out = Path(output_dir)
    out.mkdir(parents=True, exist_ok=True)

    succeeded, failed = [], []
    for prefix in prefixes:
        try:
            mapping = probe_resource(
                prefix=prefix,
                datasources=datasources,
                predicate=predicate,
                dataset_names=dataset_names,
            )
            jsonld = mapping.to_jsonld()
            outfile = out / f"{prefix}_instance_mapping.jsonld"
            outfile.write_text(json.dumps(jsonld, indent=2))
            succeeded.append(prefix)
        except Exception as exc:
            logger.error("Failed %s: %s", prefix, exc)
            failed.append({"prefix": prefix, "error": str(exc)})

    return {"succeeded": succeeded, "failed": failed}
```

### 7a. Blueprint

```python
"""Instance mapping routes — /api/mappings/*."""

from __future__ import annotations

import json
from flask import Blueprint, current_app, jsonify, request
from rdfsolve.backend.services.mapping_service import MappingService

mappings_bp = Blueprint("mappings", __name__)


def _get_svc() -> MappingService:
    return MappingService(current_app.config["DB"])


@mappings_bp.route("/", methods=["GET"])
def list_mappings():
    """List all stored mappings (strategy = 'instance_matcher')."""
    return jsonify(_get_svc().list_mappings())


@mappings_bp.route("/<mapping_id>", methods=["GET"])
def get_mapping(mapping_id: str):
    """Return a full mapping JSON-LD."""
    data = _get_svc().get_mapping(mapping_id)
    if data is None:
        return jsonify({"error": "Not found"}), 404
    return jsonify(data)


@mappings_bp.route("/probe", methods=["POST"])
def probe():
    """Probe endpoints for a bioregistry resource and return/save the mapping.

    Body:
        {
            "prefix": "ensembl",
            "predicate": "skos:narrowMatch",   // optional
            "dataset_names": ["aopwikirdf"],    // optional
            "save": true                        // optional, default true
        }
    """
    data = request.get_json(force=True)
    prefix = data.get("prefix")
    if not prefix:
        return jsonify({"error": "Missing 'prefix'"}), 400

    predicate = data.get("predicate", "http://www.w3.org/2004/02/skos/core#narrowMatch")
    dataset_names = data.get("dataset_names")
    save = data.get("save", True)

    svc = _get_svc()
    try:
        mapping_jsonld = svc.probe(prefix, predicate, dataset_names)
    except Exception as exc:
        return jsonify({"error": str(exc)}), 502

    if save:
        mid = svc.save_mapping(prefix, mapping_jsonld)
        mapping_jsonld["_saved_as"] = mid

    return jsonify(mapping_jsonld), 201


@mappings_bp.route("/<mapping_id>", methods=["DELETE"])
def delete_mapping(mapping_id: str):
    """Delete a stored mapping."""
    if not _get_svc().delete_mapping(mapping_id):
        return jsonify({"error": "Not found"}), 404
    return jsonify({"message": f"Deleted {mapping_id}"}), 200
```

### 7b. Service — `src/rdfsolve/backend/services/mapping_service.py` (NEW)

```python
"""Instance mapping service — thin Flask wrapper."""

from __future__ import annotations

import logging
from typing import Any, Optional

import pandas as pd

from rdfsolve.backend.database import Database
from rdfsolve.instance_matcher import probe_resource

logger = logging.getLogger(__name__)


class MappingService:
    """Manage instance mappings via the shared schema table.

    Mappings are stored in the same `schemas` table with
    strategy='instance_matcher'. The JSON-LD format is compatible
    with existing schema import/export.
    """

    def __init__(self, db: Database) -> None:
        self.db = db

    def list_mappings(self) -> list[dict[str, Any]]:
        """List mappings (schemas where strategy = 'instance_matcher')."""
        all_schemas = self.db.list_schemas()
        return [s for s in all_schemas if s.get("strategy") == "instance_matcher"]

    def get_mapping(self, mapping_id: str) -> dict[str, Any] | None:
        return self.db.get_schema(mapping_id)

    def save_mapping(self, prefix: str, data: dict[str, Any]) -> str:
        """Save a mapping JSON-LD. Returns the mapping_id."""
        mapping_id = f"{prefix}_instance_mapping"
        about = data.get("@about", {})
        self.db.save_schema(
            schema_id=mapping_id,
            name=about.get("dataset_name", mapping_id),
            data=data,
            endpoint="",
            pattern_count=about.get("pattern_count", 0),
            generated_at=about.get("generated_at", ""),
            strategy="instance_matcher",
        )
        return mapping_id

    def delete_mapping(self, mapping_id: str) -> bool:
        return self.db.delete_schema(mapping_id)

    def probe(
        self,
        prefix: str,
        predicate: str = "http://www.w3.org/2004/02/skos/core#narrowMatch",
        dataset_names: Optional[list[str]] = None,
    ) -> dict[str, Any]:
        """Run the instance matcher for a prefix and return JSON-LD."""
        # Load sources from the endpoints table + CSV fallback
        sources_csv = "data/sources.csv"
        datasources = pd.read_csv(sources_csv)
        mapping = probe_resource(
            prefix=prefix,
            datasources=datasources,
            predicate=predicate,
            dataset_names=dataset_names,
        )
        return mapping.to_jsonld()
```

## 10. Frontend Integration

The mapping JSON-LD intentionally uses the **same format** as schema JSON-LD (`@context` + `@graph` + `@about` + `_labels`). This means:

### 10a. Automatic compatibility

- `parseJSONLD()` already parses `@graph` nodes and their predicates into `CanonicalTriple[]`
- A mapping node like `{"@id": "aopo:Gene", "skos:narrowMatch": {"@id": "pubchem.gene:Gene"}}` will produce a `CanonicalTriple` with `predicate = "skos:narrowMatch"`, `objectType = "uri"`
- `buildTree()` will render it as an edge in the diagram
- The existing path-walking and query-composition code will include `skos:narrowMatch` edges in paths

### 10b. Visual distinction (optional, later)

## 12. Key Design Decisions

1. **Reuse the `schemas` table** — mappings are just another kind of JSON-LD document. No schema migration needed. The `strategy` column differentiates them.

2. **`STRSTARTS` not `regex`** — bioregistry URI formats are always prefixes. `STRSTARTS` is optimized by most SPARQL engines; `regex` is not.

3. **Pairwise edges, not a central hub** — when 3 datasets all have `Gene` for ensembl, we create 3 edges (A↔B, A↔C, B↔C), not a star through a synthetic node. This maps directly to SKOS semantics and is walkable by the existing path code.

4. **`to_jsonld()` matches the mined schema format** — so the existing frontend parser, database import, and startup flow all work without modification.

5. **Predicate is configurable** — default `skos:narrowMatch` can be overridden to `skos:exactMatch`, `owl:sameAs`, etc. at every level (model, API, CLI, route).

6. **Separate module, not inside miner** — `instance_matcher.py` is independent of `miner.py`. It uses `SparqlHelper` directly. This keeps concerns clean: the miner discovers schema _shape_; the instance matcher discovers cross-dataset _links_.
